# Action Parity Discipline

Use this page as the routing layer for keeping agent capabilities aligned with
user-visible product actions.

## Load Order

| Need | Reference |
| --- | --- |
| capability mapping and parity audits | `capability-map-and-audit.md` |
| designing parity-friendly tools | `tool-parity-design.md` |
| aligning runtime context with what users see | `context-parity.md` |

## Core Rules

- every important UI action should have an agent equivalent
- parity checks belong in feature delivery, not as a one-time audit
- when parity is intentionally absent, record why
