# From Primitives to Domain Tools

Use this page as the routing layer for tool evolution in agent-native systems.

## Load Order

| Need | Reference |
| --- | --- |
| why to start with primitives | `starting-with-primitives.md` |
| when to add domain tools and how to keep judgment in prompts | `domain-tool-evolution.md` |

## Core Rules

- start primitive until real patterns emerge
- add domain tools for vocabulary, guardrails, and efficiency
- optimized code paths are for hot paths, not as a substitute for agent design
