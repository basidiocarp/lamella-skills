# Swarm, Research, and Approval Patterns

Use this reference when work is self-organizing, research-led, or approval-gated.

## Swarm

- Create a pool of same-shaped tasks.
- Let workers claim tasks naturally.
- Best for many independent items of similar complexity.

## Research Then Implement

- Run research first when design choices are still open.
- Feed distilled findings into implementation tasks.

## Approval Gate

- Require a plan or architecture checkpoint before implementation.
- Use for high-risk or high-cost changes.
