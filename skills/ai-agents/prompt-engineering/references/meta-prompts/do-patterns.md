<overview>
Prompt patterns for execution tasks that create or modify real artifacts.
</overview>

<core_shape>

```xml
<objective>
Build or change {artifact}.
</objective>

<purpose>
Why this matters and what it enables next.
</purpose>

<context>
- Research or plan inputs
- Existing constraints
- Relevant file paths
</context>

<verification>
- Exact checks that prove the output is usable
</verification>
```

</core_shape>

<rules>

- Always name the output location.
- Match verification to artifact type: code, docs, config, or design.
- Reference prior research or planning artifacts when they exist.

</rules>
