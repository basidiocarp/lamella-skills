<overview>
Prompt patterns for improving an existing artifact without discarding what already works.
</overview>

<core_shape>

```xml
<objective>
Refine {artifact} based on feedback.
</objective>

<preserve>
- What must stay intact
- What should not be reformatted or re-scoped
</preserve>

<feedback>
- Specific issues to address
- Evidence or examples of the weakness
</feedback>

<output>
- Updated artifact
- Short change summary
- Version note if needed
</output>
```

</core_shape>

<rules>

- Refine by targeted changes, not broad rewrites.
- Archive or version previous outputs when the workflow depends on history.
- Make the delta explicit in the summary.

</rules>
