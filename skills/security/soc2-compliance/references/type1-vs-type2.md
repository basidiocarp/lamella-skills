# SOC 2 Type I vs Type II

## Quick Comparison

| Dimension | Type I | Type II |
|-----------|--------|---------|
| What it tests | Control design at a point in time | Control design and operating effectiveness over a period |
| Observation period | None | Usually 3 to 12 months |
| Timeline | Roughly 1 to 3 months | Roughly 6 to 15 months |
| Market signal | "Controls exist" | "Controls worked over time" |

## Start With Type I When

- this is the first SOC 2 effort
- a customer needs assurance quickly
- controls are still being put in place
- the company needs a structured checkpoint before committing to a longer observation period

## Go Straight to Type II When

- controls have already been operating for months
- enterprise customers already require Type II
- the company can support the longer timeline and evidence load
- a mature security or compliance program already exists

## Typical Upgrade Path

1. Complete a Type I assessment.
2. Fix design issues found during the readiness or Type I review.
3. Start the Type II observation period immediately after that.
4. Collect evidence continuously during the period.
5. Use the Type II audit to validate operating effectiveness.

## Planning Questions

- Which customer deals actually require Type II?
- Are controls already operating, or are they still being designed?
- Is there an owner for each recurring control?
- Is evidence collection automated enough to survive the observation period?
