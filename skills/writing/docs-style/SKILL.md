# Docs Style

Follow [writing-voice](../writing-voice/SKILL.md) for prose tone. Use this skill for documentation structure, scannability, and example quality.

## Core Rules

- be concise and task-oriented
- choose the document type before writing
- prefer real examples over abstract prose
- explain why and when, not only what
- link instead of repeating
- keep code examples copy-pasteable

## Document Types

| Type | Goal |
|------|------|
| Tutorial | teach end-to-end |
| How-to | complete one task quickly |
| Reference | look up facts consistently |
| Explanation | understand concepts and trade-offs |

## Style Guardrails

- no narrator voice
- no filler sections like “next steps”
- headings should be direct, not cute
- comments explain why, not what
- folder READMEs explain the mental model, not a filename inventory
