# Pricing Page Playbook

Use this reference when the strategy must become a pricing page that helps the right buyer choose the right plan.

## Page Job

The page should answer three questions in order:

1. Which plan is for me?
2. Is it worth the price?
3. What is the catch?

If a section does not help answer one of those, cut it.

## Core Structure

### Above the fold

- billing toggle with annual savings shown clearly
- three plan cards with obvious differentiation
- one recommended middle tier
- clear CTA per plan

### Below the fold

- full feature comparison table
- trust proof: logos, quotes, or credible customer evidence
- FAQ covering cancellation, switching plans, billing, and security
- separate enterprise CTA for custom contracts or compliance needs

## Plan Card Checklist

Each card should include:

- plan name tied to segment or use case
- clear price and billing basis
- one-line fit statement
- four to six differentiators
- CTA that matches the ask

## CTA Guidance

| Context | CTA |
|---------|-----|
| Free trial | Start free trial |
| Freemium | Get started free |
| Direct purchase | Get [plan name] |
| Sales-assisted | Talk to sales |
| Enterprise | Get a demo |

Avoid generic dead-end CTA text like `Learn more` on the pricing page itself.

## Audit Checklist

- [ ] Annual savings are visible
- [ ] The recommended tier is visually obvious
- [ ] Price notation is unambiguous
- [ ] Plan names and positioning lines are meaningful
- [ ] FAQ covers commitment, billing, and security questions
- [ ] Enterprise buyers have a distinct path forward
