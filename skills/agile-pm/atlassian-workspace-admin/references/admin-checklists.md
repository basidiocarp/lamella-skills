# Atlassian Admin Checklists

Use these checklists for routine workspace operations.

## User onboarding

- Confirm approved access request and role
- Assign products and groups through role-based mappings
- Verify SSO and MFA enrollment
- Confirm project or space access through groups, not direct user grants
- Send onboarding links or internal guidance

## User offboarding

- Audit owned Jira issues, dashboards, filters, pages, and spaces
- Transfer ownership where needed
- Remove product access and group membership
- Revoke API tokens and app authorizations
- Confirm the account is deactivated and no temporary exceptions remain

## Permission review

- Export or inspect the current scheme
- Check for direct-user grants
- Check for broad admin or delete permissions
- Confirm sensitive permissions are limited to named admin groups
- Document exceptions with owner and expiry

## Marketplace app review

- Confirm the business need
- Review vendor security posture and requested scopes
- Prefer sandbox testing before production rollout
- Record approval, rollback, and renewal owners

## Change record

Capture the request, change summary, verification evidence, rollback path, and next review date.
