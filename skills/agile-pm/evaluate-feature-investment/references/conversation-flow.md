# Feature Investment Evaluation Example

This example shows a feature request being evaluated through revenue connection, cost, and strategic value.

## Diagnostic Flow

1. Define the feature and target segment.
2. Choose the main revenue path:
   - direct monetization
   - retention
   - conversion
   - expansion
   - strategic value only
3. Capture development cost and ongoing operating cost.
4. Estimate likely return and payback.
5. Check whether strategic value changes the default recommendation.

## Example Output

For an enterprise SSO request:

1. Revenue path: expansion and deal enablement for larger accounts.
2. Cost: moderate build effort with low ongoing cost.
3. Return: supports enterprise close rate and reduces security objections.
4. Strategic value: high because it is often table stakes for the target segment.
5. Recommendation: build now if enterprise is an active segment priority; otherwise defer until the segment strategy is explicit.

## Why This Example Matters

- It separates direct ROI from strategic necessity.
- It keeps assumptions visible instead of hiding them inside a score.
- It ends with a real decision, not just a framework exercise.
