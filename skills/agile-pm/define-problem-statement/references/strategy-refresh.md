# Strategy Refresh Notes

Use these notes when problem framing is the first step in a broader product strategy refresh.

## Focus

- restate the core user problem before discussing solution themes
- connect the problem to business impact and timing
- surface the assumptions that still need discovery or validation
- use the problem statement as the anchor for later roadmap and prioritization work

## Good Signs

- the team can explain the user problem in one clear sentence
- the business reason to act now is explicit
- open questions are visible instead of hidden inside a preferred solution

## Anti-Patterns

- jumping straight to roadmap themes without a grounded problem
- turning a strategy session into a feature brainstorm
- treating stakeholder requests as proof of user need
