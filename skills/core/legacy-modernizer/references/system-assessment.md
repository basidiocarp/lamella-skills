# System Assessment

Use this page as the routing layer for legacy-system assessment work.

## Load Order

| Need | Reference |
| --- | --- |
| codebase inventory and dependency shape | `codebase-and-dependency-assessment.md` |
| technical debt and risk scoring | `debt-and-risk-assessment.md` |
| roadmap and stakeholder communication | `roadmap-and-communication.md` |

## Core Rules

- assessment should produce a modernization decision, not just diagnostics
- separate evidence gathering from rollout planning
- quantify risk and debt in terms the delivery team can act on
