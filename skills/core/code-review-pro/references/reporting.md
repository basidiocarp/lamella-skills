# Reporting

Use this page as the routing layer for code review report output.

## Load Order

| Need | Reference |
| --- | --- |
| executive summary and change framing | `executive-summary-and-change-log.md` |
| finding structure and evidence expectations | `finding-template.md` |
| test, blast radius, and historical sections | `coverage-blast-and-history.md` |
| methodology and appendix expectations | `methodology-and-appendices.md` |

## Core Rules

- findings come first and stay ordered by severity
- every claim needs file paths, evidence, and a concrete recommendation
- the report should be concise, but not at the expense of decision-useful detail
