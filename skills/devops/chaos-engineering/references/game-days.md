# Game Days

Use game days to rehearse failure handling in a controlled environment.

## Planning

Define:

- scenario and learning goal
- environment and blast radius
- participants and observers
- clear stop conditions and rollback owner

## Execution

- run one scenario at a time
- record timestamps, detection lag, mitigation actions, and communication gaps
- keep injectors and rollback steps ready before starting

## Observation

Track:

- time to detect
- time to mitigate
- whether runbooks and ownership were clear
- whether dashboards and alerts matched what operators needed

## Follow-Up

- convert findings into concrete tickets
- update runbooks and dashboards
- schedule a repeat exercise to verify fixes
