# Command Organization

Choose the simplest command layout that still makes the plugin easy to browse.

## Common Shapes

- flat: small command sets
- categorized: medium-sized plugins with clear families
- custom manifest paths: larger plugins that need explicit grouping

## Rules

- use flat layout until the command count or permissions model justifies more
  structure
- keep directory names meaningful, not abstract
- avoid deeply nested trees when a few explicit manifest paths would be clearer
