## Quick Pick

| Need | Element |
|------|---------|
| Process step or component | `rectangle` |
| Start, end, or state | `ellipse` |
| Decision | `diamond` |
| Direction or dependency | `arrow` |
| Divider or grouping line | `line` |
| Label or annotation | `text` |

## Practical Rule

Keep diagrams structurally simple. Use shapes for semantics, text for clarity,
and arrows only where the relationship matters to reading order.
