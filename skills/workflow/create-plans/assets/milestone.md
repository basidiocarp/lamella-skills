# Milestone Entry Template

Add this entry to `.planning/MILESTONES.md` when a milestone actually ships.

```markdown
## v[X.Y] [Name] (Shipped: YYYY-MM-DD)

**Delivered:** [One sentence describing what shipped]

**Phases completed:** [X-Y] ([Z] plans total)
**Git range:** `[first-commit]` → `[last-commit]`

**What changed**
- [major outcome]
- [major outcome]

**What's next:** [next milestone goal or "Project complete"]

---
```

## Create Milestones For

- v1.0 or other major shipped milestones
- meaningful feature milestones
- notable release boundaries worth preserving

## Do Not Create Milestones For

- individual phase completions
- work still in progress
- tiny bug-fix-only changes
