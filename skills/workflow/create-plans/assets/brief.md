# Brief Template

Use this for `.planning/BRIEF.md`.

## Greenfield Brief

```markdown
# [Project Name]

**One-liner:** [what this project is]

## Problem
[what needs to change]

## Users
- [primary user]
- [secondary user]

## Success Criteria
- [measurable outcome]
- [measurable outcome]

## Out of Scope
- [explicit non-goal]
- [explicit non-goal]
```

## Brownfield Brief

```markdown
# [Project Name]

## Current State (Updated: YYYY-MM-DD)

**Shipped:** v[X.Y] [Name] (YYYY-MM-DD)
**Reality:** [current product or codebase status]

## Next Goals
- [goal]
- [goal]

## Out of Scope
- [non-goal]
- [non-goal]
```

## Rules

- keep it under 50 lines where possible
- make success criteria measurable
- keep out-of-scope explicit so plans do not sprawl
