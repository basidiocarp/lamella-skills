# Roadmap Template

Use this structure for `.planning/roadmap.md`.

```markdown
# Roadmap: [Project Name]

## Overview

[One paragraph describing the product journey]

## Phases

1. [Phase name]
2. [Phase name]
3. [Phase name]

## Progress

| Phase | Plans | Status | Notes |
|------|------:|--------|-------|
| 1. [Name] | 0/1 | Not started | - |
| 2. [Name] | 0/2 | Not started | - |
| 3. [Name] | 0/1 | Not started | - |
```

## Guidelines

- keep the first roadmap to `3-6` phases
- each phase should deliver something coherent
- split a phase into multiple plans if the work exceeds roughly `7` tasks
- no time estimates
- update the progress table as the plan evolves
