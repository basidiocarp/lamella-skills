# ISSUES.md Template

Use this when `.planning/ISSUES.md` is first created.

```markdown
# Project Issues Log

## Open Enhancements
- ISS-001 — [short title]
  - discovered in: [phase/plan/task]
  - impact: [why it matters]
  - effort: [quick/medium/substantial]
  - suggested phase: [future phase or backlog bucket]

## Closed Enhancements
- ISS-000 — [title] — resolved in [phase]
```

## Rules

- auto-increment issue numbers
- record discovery context
- move resolved items to the closed section
- use the log to inform later roadmap planning
