# Component Patterns

Use this page as a compact overview of reusable component API patterns.

## Core Patterns

- compound components for shared local state
- render props when consumers need render control
- polymorphic components when semantic element choice matters
- controlled and uncontrolled APIs when the component manages state
- slots and forwarded refs for integration flexibility

## Rule

If you need the deeper design-system treatment, pair this with
`frontend/design-systems` rather than duplicating the heavier pattern catalog
