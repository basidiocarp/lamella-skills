# Component Architecture

Use this page as the routing layer for design-system component API patterns.

## Load Order

| Need | Reference |
| --- | --- |
| compound and slot patterns | `compound-and-slot-patterns.md` |
| polymorphic and headless components | `polymorphic-and-headless-components.md` |
| variants and composition patterns | `variants-and-composition.md` |

## Core Rules

- choose the lightest pattern that supports the component's real flexibility
- keep API consistency more important than novelty
- separate behavioral patterns from purely visual variants
