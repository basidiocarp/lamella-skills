# ARIA Patterns

Use this page as the routing layer for ARIA work.

## Load Order

| Need | Reference |
| --- | --- |
| ARIA roles, states, and first-principles usage rules | `aria-fundamentals.md` |
| widget patterns such as accordions, tabs, menus, and dialogs | `aria-widget-patterns.md` |
| live regions and common ARIA mistakes | `aria-live-regions-and-mistakes.md` |

## Core Rules

- native HTML beats ARIA whenever it can express the same thing
- ARIA should clarify semantics, not patch broken interaction design
- keyboard behavior and focus management matter as much as attributes
