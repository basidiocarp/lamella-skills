# Multi-Perspective Synthesis

Use this reference when an expert panel needs to collapse multiple perspectives into one decision-ready output.

## What Good Synthesis Does

Good synthesis does not restate each expert in sequence. It:

- extracts the few points that multiple experts independently support
- names real conflicts instead of smoothing them over
- reduces the action list to the items that matter now
- ends with one clear decision point or unresolved question

## Recommended Flow

1. **Read for consensus**
   Look for repeated concerns, repeated recommendations, or the same root cause showing up through different lenses.

2. **Read for conflict**
   Note where experts disagree on priority, resources, timing, or assumptions.

3. **Collapse the themes**
   Limit the synthesis to 2-3 core themes. More than that usually means the panel has not actually been synthesized.

4. **Convert analysis into action**
   Turn the shared conclusions into a short, owned list of next steps.

5. **End with one decision point**
   If leadership still has a tradeoff to choose, frame it as one clean question with options and consequences.

## Suggested Output Shape

```markdown
## What We Agree On
- [Theme 1]
- [Theme 2]
- [Theme 3]

## The Main Conflict
- [Conflict]
- [Why it exists]
- [What tradeoff it represents]

## Recommended Actions
1. [Action] — [Owner] — [Timeline]
2. [Action] — [Owner] — [Timeline]
3. [Action] — [Owner] — [Timeline]

## Decision Point
- [Question that still needs a call]
```

## Quality Checks

- [ ] Consensus points reflect multiple experts, not one loud perspective
- [ ] Disagreements are explicit
- [ ] Actions are concrete and owned
- [ ] The final decision point is singular and easy to grasp
