# Ask Questions If Underspecified

Ask questions only when ambiguity would change the implementation.

## Ask When

- multiple plausible interpretations exist
- scope, success criteria, or constraints are unclear
- safety or reversibility depends on unknowns

## Do Not Ask When

- a quick low-risk read answers the question
- the request is already specific enough to proceed

## Rules

- ask the minimum number of questions that removes the ambiguity
- prefer short, answerable questions over open-ended interviews
- if the user wants you to proceed anyway, state assumptions before acting
- restate the chosen interpretation once answers arrive
