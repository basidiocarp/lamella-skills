# Coding Style

Use this rule for readability and structure. Keep project-specific conventions in [coding-standards.md](./coding-standards.md).

## Rules

- prefer clear names over clever abbreviations
- use early returns to keep control flow shallow
- keep files and functions cohesive
- extract helpers only when the new name improves clarity
- comment on intent, constraints, or tradeoffs; not obvious behavior
- let the formatter and linter own layout
