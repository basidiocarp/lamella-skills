# Git Workflow

## Commit Message Format
```text
<type>: <description>
```

Types: feat, fix, refactor, docs, test, chore, perf, ci

## Pull Request Workflow

- review the full branch diff, not only the last commit
- write a summary that matches the real scope
- include validation performed and notable follow-up work
- keep git process guidance here; keep planning and review guidance in [development-workflow.md](./development-workflow.md)
