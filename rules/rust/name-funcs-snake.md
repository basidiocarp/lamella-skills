# name-funcs-snake

> Use `snake_case` for functions, methods, variables, and modules

Follow standard Rust casing for value-level names.

## Prefer

- `snake_case` for functions, locals, modules, and methods
- names that describe behavior or role clearly
- local consistency over novelty

## Avoid

- CamelCase or mixedCase in ordinary Rust value names
- abbreviations that obscure intent
