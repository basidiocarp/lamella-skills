# name-variants-camel

> Use `UpperCamelCase` for enum variants

Enum variants should follow the same CamelCase convention as types.

## Prefer

- `UpperCamelCase` variant names
- variant names that describe the state or case clearly
- consistency across all variants in the enum

## Avoid

- snake_case or screaming case variants
- variants distinguished only by cryptic abbreviations
