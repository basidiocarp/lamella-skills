# name-types-camel

> Use `UpperCamelCase` for types, traits, and enum names

Follow standard Rust casing for type-level items.

## Prefer

- `UpperCamelCase` for structs, enums, traits, and type aliases
- names that reflect domain meaning clearly
- consistent casing for related items across the crate

## Avoid

- snake_case or mixedCase for type names
- abbreviations that make type meaning harder to read
